Java Collections - Simple Notes (With Types)

1) List (Order, Allows Duplicates)

Type: List → Interface | ArrayList → Class

List stores items in order and allows duplicates.

List list = new ArrayList<>();
list.add("Apple");
list.add("Banana");
list.add("Apple");
System.out.println(list);

2) Set (No Duplicates)

Type: Set → Interface | HashSet → Class

Set stores unique items. No duplicates allowed.

Set set = new HashSet<>();
set.add("Apple");
set.add("Banana");
set.add("Apple");
System.out.println(set);

3) TreeSet (Sorted Set, No Duplicates)

Type: Set → Interface | TreeSet → Class

TreeSet stores unique items in sorted order.

Set set = new TreeSet<>();
set.add("Banana");
set.add("Apple");
set.add("Orange");
System.out.println(set); 
// Output: [Apple, Banana, Orange]

4) Queue (First In, First Out)

Type: Queue → Interface | LinkedList → Class

Queue works like a line. First added = first removed.

Queue q = new LinkedList<>();
q.add("A");
q.add("B");
System.out.println(q.remove()); // A

5) HashMap (Key → Value, No Order)

Type: Map → Interface | HashMap → Class

HashMap stores key-value pairs without order.

Map map = new HashMap<>();
map.put("Apple", 50);
map.put("Banana", 30);
map.put("Apple", 60); 
System.out.println(map);

6) TreeMap (Sorted Map)

Type: Map → Interface | TreeMap → Class

TreeMap stores key-value pairs in alphabetically sorted order.

Map map = new TreeMap<>();
map.put("Banana", 30);
map.put("Apple", 50);
map.put("Orange", 40);
System.out.println(map);
