Object Class in JavaScript

The Object class in JavaScript is the base (root) class from which all other objects inherit. Every object in JavaScript directly or indirectly inherits properties and methods from Object.prototype. It provides fundamental functionality to all objects.

Why Object Class is Important?

  • It acts as the parent class of all objects.
  • Provides common built-in methods used by all objects.
  • Forms the base of the prototype chain.

Creating an Object Using Object Class

1. Using new Object()

      let person = new Object();
      person.name = "Yogita";
      person.age = 20;

2. Using Object Literal (recommended)

      let person = {
        name: "Yogita";
        age: 20
      };

Common Methods of the Object Class

1. Object.keys() — returns keys of an object

      Object.keys(person); // ["name", "age"]

2. Object.values() — returns values

      Object.values(person); // ["Yogita", 20]

3. Object.entries() — returns key-value pairs

      Object.entries(person);
      // [["name","Yogita"], ["age",20]]

4. hasOwnProperty() — checks if property exists

      person.hasOwnProperty("name"); // true

5. Object.freeze() — makes object unchangeable

      Object.freeze(person);

6. Object.assign() — copies object properties

      let newObj = Object.assign({}, person);

Prototype Link (Object Prototype)

      let obj = {};
      console.log(obj.__proto__ === Object.prototype); // true

This shows all objects inherit properties and methods from the Object class through the prototype chain.

Compilation and Decompilation

Compilation

Compilation is the process of converting high-level source code into machine code that the computer can understand. It is performed by a compiler. The entire program is compiled at once, and errors are reported after compilation.

  • Converts high-level code → machine code
  • Faster execution since compiled code is ready to run
  • Output files like .exe, .class, .o
  • Examples: GCC compiler, javac for Java
      Source Code → Compiler → Machine Code

Decompilation

Decompilation is the reverse of compilation. It converts machine code or bytecode back into readable high-level code. This process is performed by a decompiler.

  • Machine code → high-level code
  • Used for debugging, analysis, code recovery
  • Output is not always exactly the original code
  • Examples: JD-GUI, JAD, APK decompilers
      Machine Code / Bytecode → Decompiler → High-Level Code

Difference Between Compilation & Decompilation

Feature Compilation Decompilation
Direction High-level → Machine code Machine code → High-level
Tool Used Compiler Decompiler
Purpose To run the program To understand or recover code
Accuracy Exact conversion Not always exact
Examples gcc, javac JD-GUI, JAD

Enhanced For Loop (For-Each Loop)

The Enhanced for loop, also known as the for-each loop, is used to iterate over arrays or collections in a simple and readable way. Unlike the traditional for loop, it does not require using an index.

Syntax (Java)

      for (datatype variable : arrayOrCollection) {
          // statements
      }

It takes each element from the array or collection, stores it in a temporary variable, and runs the loop for every element.

Example in Java

      int numbers[] = {10, 20, 30, 40};
      
      for (int n : numbers) {
          System.out.println(n);
      }

Example with Strings

      String names[] = {"Yogita", "Siddhi", "Saanvi"};
      
      for (String name : names) {
          System.out.println(name);
      }

Advantages

  • No index required
  • Cleaner and more readable code
  • Prevents index-out-of-bound errors

Limitations

  • Index cannot be accessed directly
  • Cannot modify elements directly in some cases
  • Works only with arrays and iterable collections

Similar Concept in JavaScript (for...of Loop)

      let arr = [10, 20, 30];
      
      for (let item of arr) {
          console.log(item);
      }

The enhanced for loop simplifies iteration and improves readability, making it a preferred choice when index values are not required.

Legacy Class

A Legacy Class refers to an old or outdated class that was created in earlier versions of a language or framework. These classes may still work today, but they are not recommended for new development because modern alternatives offer better performance, security, and features.

Legacy classes are usually kept for backward compatibility so older programs do not break when the language or platform updates.

Characteristics of Legacy Classes

  • Old-style classes from earlier versions of a language
  • May lack modern features and improvements
  • Kept for backward compatibility
  • Not recommended for new development
  • Often replaced by newer, more efficient classes

Example (Java Legacy Classes)

Some classes in Java are considered legacy because they were part of early Java versions before Collections Framework was introduced.

      // Legacy class example in Java
      import java.util.Vector;
      
      Vector list = new Vector<>();
      list.add(10);
      list.add(20);
      System.out.println(list);

Better modern alternative: ArrayList

      import java.util.ArrayList;

      ArrayList list = new ArrayList<>();
      list.add(10);
      list.add(20);
      System.out.println(list);

Legacy classes are useful for maintaining old projects, but modern applications should use updated and more efficient classes.

Array of Objects

An Array of Objects is an array where each element is an object. This structure is useful when we need to store multiple records, each containing multiple properties (like name, age, marks, etc.).

Arrays help store multiple objects, and objects help store multiple data fields — making this combination powerful for real-world data.

Example (JavaScript)

      let students = [
        { name: "Yogita", age: 20, marks: 85 },
        { name: "Siddhi", age: 19, marks: 90 },
        { name: "Saanvi", age: 21, marks: 78 },
      ];
      
      console.log(students[0].name); // Output: Vedika

Looping Through Array of Objects

      // Using for loop
      for (let i = 0; i < students.length; i++) {
        console.log(students[i].name + " - " + students[i].marks);
      }
      
      // Using for...of loop
      for (let s of students) {
        console.log(s.name, s.age, s.marks);
      }

Example (Java)

      // Java class
      class Student {
          String name;
          int age;
      
          Student(String n, int a) {
              name = n;
              age = a;
          }
      }
      
      public class Main {
          public static void main(String[] args) {
      
              // Array of objects
              Student[] arr = {
                  new Student("Yogita", 20),
                  new Student("Siddhi", 19),
                  new Student("Saanvi", 21),
              };
      
              for (Student s : arr) {
                  System.out.println(s.name + " " + s.age);
              }
          }
      }

Array of objects is widely used when storing structured data such as student lists, employee records, product details, and more.

Generics

Generics allow us to create classes and methods that can work with any data type, while still providing type safety. This means Java checks incorrect data types during compile time and prevents errors.

Generics make code reusable, safe, and easy to understand.

Simple Generic Class Example

      class Box {   // T means any type
          T item;
      
          void set(T item) {
              this.item = item;
          }
      
          T get() {
              return item;
          }
      }
      
      public class Main {
          public static void main(String[] args) {
      
              Box b1 = new Box<>();
              b1.set(10);
      
              Box b2 = new Box<>();
              b2.set("Hello");
      
              System.out.println(b1.get());  // 10
              System.out.println(b2.get());  // Hello
          }
      }

Explanation

  • T means "type" — it can become Integer, String, Float, etc.
  • Box<Integer> → T becomes Integer
  • Box<String> → T becomes String
  • Prevents wrong inputs at compile time

Generic Method Example

      public  void display(T value) {
          System.out.println(value);
      }
      
      display(100);        // Integer
      display("Yogita");   // String

Generics ensure type safety and remove the need for manual type casting, making Java programs safer and cleaner.

Wrapper Class

A Wrapper Class in Java is a class that converts a primitive data type into an object. Java provides a wrapper class for every primitive type. These classes “wrap” the primitive value inside an object so it can be used in object-based features like Collections, Generics, etc.

Why Do We Need Wrapper Classes?

  • To store primitive values in objects (e.g., in ArrayList)
  • To use features like Collections and Generics (which only work with objects)
  • To use utility methods present in wrapper classes
  • For automatic conversion (autoboxing & unboxing)

Primitive Type → Wrapper Class

Primitive Type Wrapper Class
byteByte
shortShort
intInteger
longLong
floatFloat
doubleDouble
charCharacter
booleanBoolean

Example: Converting Primitive to Object (Boxing)

      int a = 10;
      
      // Boxing (manual)
      Integer obj1 = Integer.valueOf(a);
      
      // Autoboxing (automatic)
      Integer obj2 = a;
      
      System.out.println(obj1); // 10
      System.out.println(obj2); // 10

Explanation: Primitive int is converted into its object type Integer. Java automatically converts when needed.

Example: Converting Object to Primitive (Unboxing)

      Integer x = 50;
      
      // Unboxing (manual)
      int y = x.intValue();
      
      // Auto-unboxing (automatic)
      int z = x;
      
      System.out.println(y); // 50
      System.out.println(z); // 50

Explanation: Object Integer is converted back to primitive int.

Difference Between Primitive and Wrapper Class

Primitive Wrapper Class
Stores simple values Stores values as objects
Cannot be used in Collections Can be used in Collections (ArrayList, HashMap)
Faster performance Slower because of object creation
No utility methods Provides useful methods (parseInt, valueOf)
Default values like 0, false Default value is null

Example: Wrapper Class Useful in Collections

      // int cannot be used directly in ArrayList
      ArrayList list = new ArrayList<>();
      
      list.add(10);   // autoboxing
      list.add(20);
      
      System.out.println(list); // [10, 20]

Explanation: ArrayList only stores objects. So int values are automatically converted into Integer objects.

Wrapper classes make Java more flexible by combining the power of primitive speed with object-oriented features.
